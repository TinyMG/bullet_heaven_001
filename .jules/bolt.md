## 2024-04-08 - Target Caching & Squared Distance Optimization
 **Learning:** In Godot 4, calling `get_tree().get_nodes_in_group()` every frame (e.g., in `_physics_process` for homing projectiles) causes major performance bottlenecks due to continuous array allocation. Additionally, using `distance_to()` requires expensive square root calculations, compounding the issue when checking many entities. Finally, `not is_instance_valid(_target)` evaluates to `true` if `_target` is `null`, which can cause unnecessary targeting queries if not handled properly.
 **Action:** Use interval-based targeting (e.g., 0.2s) instead of per-frame querying. Cache the target and check its validity with `_target != null and not is_instance_valid(_target)`. Always prefer `distance_squared_to()` for distance comparisons in hot loops.
