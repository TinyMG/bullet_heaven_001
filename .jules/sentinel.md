## 2024-05-18 - JSON Deserialization Type Validation
**Vulnerability:** The application was parsing local JSON files (`user://save_data.json` and `user://settings.json`) and directly casting the results to strongly-typed variables or assuming specific field types (like arrays or floats) without verifying them first. An attacker or end-user modifying these local files to have unexpected types (e.g., an integer instead of a dictionary) would cause the application to crash or throw type-casting errors.
**Learning:** In Godot 4.x, when using `JSON.parse()`, the `json.data` variant needs strict validation. Assuming a Dictionary or Array format based on expected structure is unsafe for user-controlled local files.
**Prevention:** Always use `typeof()` checks (e.g., `typeof(json.data) == TYPE_DICTIONARY`) before accessing fields, and validate each individual field's type (e.g., `typeof(data["completed_nodes"]) == TYPE_ARRAY`) before casting or assigning it to game state variables.
